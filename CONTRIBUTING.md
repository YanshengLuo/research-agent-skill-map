# Contributing

Contributions should make the comparison more accurate, current, and useful without turning it into a ranking.

## Suggest a repository

Open a repository-suggestion issue and include:

- the public repository URL;
- its primary research category;
- the task or gap it covers;
- links to upstream documentation supporting the proposed description; and
- any conflict of interest.

Candidate repositories should be open source, publicly documented, relevant to research-agent skills, and distinct enough to improve the selector or comparison.

## Correct or improve a description

Use the correction issue form for outdated or inaccurate text. Quote the guide's current wording, provide the corrected wording, and link to current upstream evidence.

Pull requests are welcome for focused changes to the README, structured registry, visual accessibility, or maintenance workflow. Keep descriptions neutral, concise, and paraphrased. Do not add star-based rankings, private information, personal endorsements, or unverified claims.

## Review checklist

Before submitting a pull request:

1. Update `last_reviewed` or `last_verified` only for sources actually checked.
2. Keep `README.md` and `data/repositories.yml` consistent.
3. Confirm that Markdown links resolve.
4. Validate YAML and SVG changes.
5. Check that every upstream project is credited and linked to its original repository.
6. Do not add or change this repository's license without explicit maintainer approval.
